# 4-1단계 — Mapper XML 쿼리 품질 검증

Mapper XML을 생성한 경우, **즉시 아래 절차로 쿼리 품질을 자체 검증**한다. MyBatis SQL 작성 컨벤션(스키마 접두사·대문자·동적 SQL·함수 시그니처 등)은 `.claude/docs/guideline/guide-mybatis.md` 와 `.claude/rules/base-rule.md` _DB 쿼리 공통 규칙_ 을 따른다.

---

## 4-1-1. 사용 테이블·스키마 메타데이터 조회

1. 생성한 쿼리에서 **사용 테이블 추출** (FROM, JOIN, INSERT INTO, UPDATE — 서브쿼리 내 테이블도 빠짐없이)
2. `mcp__postgres__query`로 각 테이블의 **인덱스·PK/UNIQUE·컬럼 정보(`data_type` 포함)** 조회 (병렬 실행 권장)
 - 조회 SQL은 루트 `CLAUDE.md`의 **Database Schema** 섹션 참조
3. **`data_type` 명시 의무**: SQL 함수(`TO_CHAR`, `TO_DATE`, `CAST`, `||` concat, 산술 연산자 등)에 입력으로 들어가는 **모든 컬럼의 실제 `data_type`을 본 단계에서 반드시 확인**한다. 4-1-2 정적 검증 표의 _함수 시그니처 호환성_ 행이 이 메타데이터를 입력으로 사용한다.

---

## 4-1-2. 정적 쿼리 검증 항목

> **검증 룰의 단일 출처는 가이드**. 본 표는 _에이전트가 어떤 가이드 항목을 어떤 시점에 적용하는가_ 의 매핑만 명시한다.

| 검증 영역| 체크 기준 (단일 출처)|
| ---------------------------------------------------------------------------------------------------------| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 작성 규칙 (스키마 접두사 / 대문자 / `SELECT *` 금지)| `base-rule.md §8` _DB 쿼리 공통 규칙_ + `guide-mybatis.md §2·§3`|
| 자가 점검 8개 항목 (쿼리 목적·입출력·JOIN 구조·WHERE·동적 SQL·사용자 정의 함수·인덱스 활용·최적화 포인트)| `guide-mybatis.md §4`|
| 인덱스 활용 패턴 (Eq / Range / Seq Scan 판정)| `guide-mybatis.md §7`|
| **함수 시그니처 ↔ 컬럼 `data_type` 호환성** (에이전트 고유 — 가이드 미수록)| **모든 SQL 함수(`TO_CHAR`/`TO_DATE`/`CAST`/`SUBSTRING`/`LPAD`/`concat(\|\|)`/산술 등)의 입력 인자 타입이 PostgreSQL 함수 오버로드 표에 존재하는가**. 특히 _문자열 컬럼에 `TO_CHAR` 직접 호출 금지_ (`TO_CHAR(varchar)` 시그니처 부존재 → 런타임 `function does not exist` 에러). 4-1-1에서 조회한 `data_type` 메타데이터를 기반으로 함수 호출별 매트릭스 검증. 변환이 필요하면 명시 캐스팅(`TO_DATE(varchar,'YYYYMMDD')` 후 `TO_CHAR(date,'YYYY.MM.DD')` 등) 사용.|

---

## 4-1-3. 동적 SQL 분기별 인덱스 분석 (필수)

생성한 쿼리에 `<if>` / `<choose>` / `<foreach>` / `<where>` / `<set>` / `<trim>` 등 동적 태그가 있으면 **각 분기별 실행 SQL을 도출하여** `guide-mybatis.md §5` _동적 SQL 분기별 인덱스 활용_ 의 분석 항목을 모두 적용한다 (분기별 인덱스 활용 차이 / `<foreach>` 100건 이상 임시 테이블 JOIN 전환 등).

---

## 4-1-4. 사용자 정의 함수(`FN_*`) 영향 분석

생성한 쿼리에 `FN_*` 사용자 정의 함수 호출이 있으면 `guide-mybatis.md §6` _사용자 정의 함수 호출 시 고려 사항_ 의 4개 점검 항목(호출 위치·호출 빈도·내부 I/O·캐싱 가능성)을 모두 적용한다. 함수 소스를 확인할 수 없으면 _"내부 로직 미확인"_ 으로 표시하고 5단계 보고에 포함한다.

---

## 4-1-5. 심층 분석 권장 케이스

쿼리가 다음 중 하나에 해당하면 본 자체 검증을 통과하더라도 **5단계 보고에 *"심층 분석 권장"* 표시를 명시**하고, 사용자에게 별도 쿼리 분석(EXPLAIN ANALYZE / DBA 검토 등)을 권유한다:

- 동적 SQL 분기 ≥ 3개
- 사용자 정의 함수가 WHERE 절에 사용
- JOIN 테이블 ≥ 4개
- 대량 데이터 조회 (LIMIT 없이 전체 SELECT)

검증 결과 수정이 발생하면 **수정 전/후를 5단계 보고에 포함**한다.
