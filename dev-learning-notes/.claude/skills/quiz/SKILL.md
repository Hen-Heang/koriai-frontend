# /quiz [topic] — Quick Drill

A short drill, 5-10 minutes. Not a full session.

## Steps

1. Read `PROGRESS.md`.
2. Pick questions:
   - If a topic was given: 3-5 questions on that topic, easy → hard.
   - If no topic: pull from "Weak spots" first, then recent topics.
3. Ask ONE question at a time. Wait for the answer.
4. Grade each: PASS / WEAK / FAIL — honestly.
   - WEAK or FAIL: give a small hint, make him re-answer.
     Still do not reveal the answer until he has tried twice.
5. At the end: show his score, update `PROGRESS.md` quiz history
   and weak spots.

## Question style

- Real interview phrasing ("What is the difference between...",
  "What happens if...", "Why would a bank...")
- Mix theory and "what does this code print" snippets (under 10 lines)
- Banking-flavored when possible
