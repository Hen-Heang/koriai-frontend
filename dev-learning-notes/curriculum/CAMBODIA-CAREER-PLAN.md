## My one-line edge

Java/Spring Boot **+ SQL + English + Korean**. Korean is the
door-opener at Korean-owned banks (Korean Desks, Korean HQ teams,
Korean system integrators). The hard requirements are still
**Java/Spring + SQL + English** — Korean is the differentiator on top.

## Chosen target: Korean banks first

I optimize for the **Korean-owned bank cluster** (best use of my
Korean). These lean toward **Oracle + PL/SQL and core-banking**
systems, so I master **Oracle PL/SQL deepest**, keep **PostgreSQL**
as a strong second.

## What Cambodian banks actually run (so I'm not surprised)

- **Core banking vendors:** **Temenos** (T24/Transact) is the most
  common — ACLEDA, Canadia, Amret, AMK, LOLC, Woori, Cambodia Post.
  **Oracle FLEXCUBE** — KB PRASAC, Prince, Hattha/HKL, Wing.
- **Databases:** Oracle + **PL/SQL** dominate core-banking shops;
  PostgreSQL on digital/mobile teams; MS SQL (T-SQL) at some Korean
  banks (PPCBank).
- **Payments:** **Bakong** (NBC's blockchain retail payment system,
  built with Soramitsu on Hyperledger Iroha, launched 2020) and
  **KHQR** (national interoperable QR standard). Integration with
  Bakong/KHQR is now table-stakes for any bank's digital channel.

## Skill gaps this plan must close

1. **Oracle PL/SQL** depth (procedures, packages, cursors, triggers) —
   primary DB target. → ROADMAP Phase 4.
2. **Spring Boot for banking**: microservices, REST design,
   Spring Security, JPA *and* MyBatis (MyBatis maps well to
   PL/SQL-heavy shops), Kafka/RabbitMQ, Docker/CI-CD. → Phases 6–7.
3. **Bakong/KHQR-aware portfolio project**: Spring Boot payment
   service + QR-payment flow + Next.js front end. Proves domain fit.
   → Phase 7 capstone.
4. **Interview English** + keep Korean technical fluency
   (TRANSLATION MODE) for HQ/SI liaison value.

## Salary context (market estimate, not bank-specific)

Per Loma Technology "Cambodia Salary Breakdown 2026": entry
~$400–800/mo; experienced/specialized ~$1,000–2,000/mo; mobile /
cybersecurity / data / international-firm roles above that.
Foreign-owned firms pay ~20–30% above local; fintech/banking carries
a premium. A mid-to-senior bank backend engineer targets the upper
band — Korean-owned + core-banking skills cluster at the top.

## Caveats (don't over-trust the research)

- ABA & Shinhan Cambodia don't publish full stacks; some inferences
  come from sister-country units, not local confirmation.
- Hana Bank Cambodia IT hiring is unconfirmed.
- Korean is an *advantage*, not a stated requirement in the actual
  Cambodia developer ads — English + Java/SQL are the hard ones.
- Salary figures and job deadlines are time-sensitive; re-verify
  before applying.

## Benchmarks that would change the plan

- Traction at **ABA/Wing on pure merit** → de-prioritize the Korean
  angle, optimize for engineering culture/pay (pivot DB toward
  PostgreSQL — see `ROADMAP.md` Phase 4 note).
- A Korean bank offering a **management / HQ-liaison** track → the
  Korean premium outweighs a slightly higher local-firm salary.
- If **PL/SQL roles dominate** my shortlist → invest more there before
  arriving; if **PostgreSQL/Node digital** roles dominate → pivot.
